# Obsidian Prospect Engine

Production lead generation, CRM, outreach, client management, payment tracking, and commission platform for **Obsidian Systems LLC**.

The app helps Obsidian Systems find local businesses with weak online presence, save and qualify leads, generate outreach drafts, manage campaigns, onboard clients, track Square invoices/subscriptions, and calculate lead-generator commissions.

```text

```

Repository:

```text

```

## Project Overview

Obsidian Prospect Engine is an internal operating system for local-business website sales.

It combines:

- Google Places lead discovery
- Website audit and lead scoring
- CRM and saved lead management
- Email/SMS outreach draft generation
- Approval-based outbound outreach
- Campaign organization
- Client and invoice tracking
- Square payment and retainer synchronization
- Commission tracking for lead generators
- Export tools for leads and CRM data

The platform is private by default and requires authenticated access.

### Square Payments and Retainers

The app is designed to use Square-hosted payment tools. It does **not** store raw card numbers, CVV, or full payment credentials.

Stored Square references:

- Square customer ID
- Square card/payment method reference if provided by Square
- Square subscription ID
- Square invoice ID
- Square payment ID
- Square invoice URL
- payment status
- payment history timeline

Webhook endpoint:

```text
https://prospect.obsidian-systems.tech/api/webhooks/square
```

Webhook processing:

- verifies Square webhook signature
- records webhook event payload
- de-duplicates webhook events by Square event ID
- matches clients by Square customer, invoice, subscription, or payment references
- updates local invoice status (`DRAFT`, `SENT`, `VIEWED`, `PAID`, `FAILED`, `CANCELED`)
- updates matching client payment status
- updates last/next payment dates
- records payment history
- creates in-app notifications
- creates one commission record per confirmed Square payment

Client billing setup now supports:

1. converting a saved lead into a linked client
2. entering a manual upfront website build amount
3. selecting a retainer tier
4. sending a Square-hosted invoice for the upfront build when Square env vars are configured
5. creating/linking a Square subscription for recurring retainers when Square catalog plan variation IDs are configured
6. storing Square customer, order, invoice, payment, and subscription references

Retainer tiers:

- `PROMO_100` - $100/month
- `STANDARD_200` - $200/month
- `CUSTOM` - local amount support; requires a Square plan variation ID before live subscription creation

### Commission Tracking

Default commission:

```text
10%
```

Commission statuses:

- `PENDING`
- `READY_TO_PAY`
- `APPROVED`
- `PAID`
- `VOIDED`

When Square confirms a retainer payment:

1. client is identified by Square customer, invoice, or subscription reference
2. assigned/onboarding lead generator is identified
3. retainer commission is calculated
4. commission is created with `READY_TO_PAY`
5. notifications are created for the lead generator and SUPER_ADMIN

Super Admin can approve or mark payouts paid from the commissions workflow. The app does not auto-transfer funds yet; payout provider/reference fields are stored for future automation.

Important service layers:

## Square Webhook Setup

Create a webhook subscription in the Square Developer Dashboard.

Webhook URL:

```text
https://prospect.obsidian-systems.tech/api/webhooks/square
```

Configure events for invoices, payments, and subscriptions. Recommended event categories:

```text
invoice.*
payment.*
subscription.*
```

Set these env vars in Vercel:

```bash
SQUARE_WEBHOOK_SIGNATURE_KEY=
SQUARE_WEBHOOK_URL=https://prospect.obsidian-systems.tech/api/webhooks/square
SQUARE_ACCESS_TOKEN=
SQUARE_LOCATION_ID=
SQUARE_ENVIRONMENT=production
SQUARE_PROMO_RETAINER_PLAN_VARIATION_ID=
SQUARE_STANDARD_RETAINER_PLAN_VARIATION_ID=
```

Signature verification uses Square's HMAC-SHA256 method with the webhook signature key, exact notification URL, and raw request body. The app compares the computed signature with the `x-square-hmacsha256-signature` header.

Important:

- The notification URL in Square must exactly match `SQUARE_WEBHOOK_URL`.
- Use HTTPS in production.
- Keep the signature key server-side only.
- Do not parse/modify the raw body before signature verification.

### Square Subscription Plan Setup

Square subscription creation requires catalog subscription plan variation IDs. Create the $100/month promo plan and $200/month standard plan in Square, then copy their plan variation IDs into:

```bash
SQUARE_PROMO_RETAINER_PLAN_VARIATION_ID=
SQUARE_STANDARD_RETAINER_PLAN_VARIATION_ID=
```

Without those IDs, the app still saves the local client billing setup and sends the upfront invoice when Square invoice env vars are available, but recurring subscription setup reports as unavailable instead of crashing.

## Security Notes

- Session cookies are HTTP-only and signed with `SESSION_SECRET`.
- Middleware blocks unauthenticated access.
- API keys are server-side only.
- OpenAI, Twilio, Resend, and Square all fail gracefully when optional env vars are missing.
- Direct deposit details are never returned raw to the client.
- Bank details are encrypted server-side when `PAYOUT_ENCRYPTION_KEY` is configured.
- Raw card numbers and CVV are never stored.
- Square webhook signatures are verified before processing.
- Lead generator data is scoped by owner/visibility rules.

## Troubleshooting

### Build succeeds locally but production errors on database fields

Run:

```bash
npm run db:push
```

Then redeploy:

```bash
npx vercel --prod --yes
```

### Login fails with session secret error

Set:

```bash
SESSION_SECRET=your-32-character-minimum-secret
```

### Lead search fails

Check:

```bash
GOOGLE_PLACES_API_KEY=
```

Ensure Places API (New) and Geocoding API are enabled.

### Square webhook returns invalid signature

Check:

- `SQUARE_WEBHOOK_SIGNATURE_KEY`
- `SQUARE_WEBHOOK_URL`
- exact URL in Square dashboard
- production vs sandbox app credentials

### Square invoice sends but subscription setup is unavailable

Check:

- `SQUARE_PROMO_RETAINER_PLAN_VARIATION_ID`
- `SQUARE_STANDARD_RETAINER_PLAN_VARIATION_ID`
- matching sandbox vs production Square environment

### New schema fields missing in production

After schema changes, run:

```bash
npm run db:push
```

Then redeploy with Vercel.

### Direct deposit values are not encrypted

Set:

```bash
PAYOUT_ENCRYPTION_KEY=
```

Use either:

- 32-byte base64 key
- 64-character hex key

Without the key, the app stores only masked last-four metadata.
